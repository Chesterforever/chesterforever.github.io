面向3D打印机的OS编写

前言

个人3D打印OS固件项目，参考了目前市面主流的klipper框架，使用正点原子stm32mp157开发板，尽可能完成3D打印机所需的功能在这块开发板的适配，持续更新中……

思路整理

硬件选择了较为熟悉的ST系列的STM32MP157，核心配置是异构cortex-A7*2+cortex-M4，可以同时运行linux（A核）和实时OS系统（M核），而且资料文档很丰富，软件生态很完整。其中ST厂商将两个cortex-A核分为了安全核和非安全核，暂时不知道怎么处理，开发时遇到问题再看。

linux内核和Uboot使用ST的出厂源码，linux文件系统目前使用正点原子的rootfs，省去前期环境搭建的时间，后面如果有时间再看看怎么优化。（文件系统尝试了一下使用buildroot搭建，发现太多坑了，即便有图形化配置还是存在很多配置冲突的奇怪问题，比如设置开机密码导致的权限问题）

首先分析一个3D打印机需要哪些模块化的功能，需要哪些外设：

  • x个串口用于步进电机的控制和加热头的控制

  • 温度检测有两个方案：一个是热敏电阻+ADC，一个是温度传感器，前者做不到3D打印机所需的精确温控，所以采用温度传感器方案，需要看厂商给的信号协议，

  • 散热风扇：同样看厂商给的信号协议，

  • 摄像头模块OV5640

  • 异核间通信

  • LCD屏幕驱动

  • 电容多点触摸FT5426模块驱动

  • 红外模块（用于安全监测）

  • SDmmc模块

  • USB驱动(OTG,HUB)

设备树

1.pinctrl修改

stm32mp1自带液晶屏幕接口LTDC，可以用于连接RGB LCD的屏幕，ST厂商已经在stm32mp15-pinctrl.dtsi中给出了ltdc_pins_a和ltdc_pins_b，这是两套预设的物理引脚配置，这里用速率更高的ltdc_pins_b。查看模块原理图发现，使用的LCD引脚与ST给出的代码配置一致，所以这里不用修改。

2.LTDC接口修改

在stm32mp151.dtsi中查找LTDC，发现已经给出了一部分初始配置

	ltdc: display-controller@5a001000 {
		compatible = "st,stm32-ltdc";
		reg = <0x5a001000 0x400>;
		interrupts = <GIC_SPI 88 IRQ_TYPE_LEVEL_HIGH>,
			     <GIC_SPI 89 IRQ_TYPE_LEVEL_HIGH>;
		clocks = <&rcc LTDC_PX>;
		clock-names = "lcd";
		resets = <&rcc LTDC_R>;
		status = "disabled";
	};

linux源码下documentation/devicetree/bindings/display/st,stm32-ltdc.txt中给出了相关的配置资料，会发现LTDC还支持MIPI屏幕

在st的基础上进行了追加节点的修改，这里status改为okay后上面👆那段代码的status就不用改了，因为linux的驱动算法会覆盖

对于dts和dtsi的嵌套问题倒是在github上找到一些脚本，能简单地输出文件结构树状图

多点触摸屏I2C驱动

使用FT5426作为触摸屏控制IC，以linux下的I2C设备驱动框架作为主框架，还要用到中断驱动框架和input子系统框架，使用MT协议的TypeB方式进行I2C传输

当触摸屏持续被触摸时会一直触发中断，但是I2C最大速度400khz，会花费大量CPU时间，但是触摸对于实时性要求没有那么高，所以应该使用devm_request_threaded_irq申请线程化中断，这种中断可以被赋予优先级，从而保证其他线程可以正常运行

设备树修改

可以看到ST厂商已经写好了I2C的pinctrl相关代码，并且PH4和PH5与我们的期望相符

设备树的pinctrl配置分成了两部分，将提供者系统和消费者系统分离

我们所需做的就是在I2C设备树下追加节点
